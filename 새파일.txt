Git. Version Control System / 소스코드 변경 이력을 기록,관리
Git은 내 PC에 설치하는 로컬 관리도구. GitHub은 이것들을 클라우드에서 관리하고 활용하는 서비스

Git 설치
git-scm.com
설치 후 git --version으로 확인.

GitHub에서 새 저장소 생성 (예: endeavour-project), 옵션: Add a README file 체크 → 기본 설명 파일 자동 생성.


로컬과 연결
1. 프로젝트 폴더로 들어감
  cd D:\MyProject
2. git 초기화(레포지토리 생성)
  git init
3. github 원격 저장소 연결
  git remote add origin https://github.com/아이디/저장소명.git
4. git add, git commit으로 버전 기록, git push로 GitHub와 동기화

기본 워크플로우
1. 작업파일 추가
  git add .
2. 커밋(Commit)
  git commit -m "전략 엔진 기본 구조 작성"
3. GitHub 업로드
  git push origin main

복구 전략 : 실수했을 때 git log로 기록 확인 후,
  git checkout <commit_id>


*📌 최소 Git Flow 실전 예시*

1. 준비

Git 설치 완료 (한 번만 하면 됨)

VS Code + GitHub 계정 연동

2. 저장소 처음 가져오기
git clone https://github.com/아이디/저장소명.git
cd 저장소명


→ GitHub에 있는 저장소를 내 PC로 복제

3. 새로운 기능 작업하기 (브랜치 생성)
git checkout -b feature/double-bottom-strategy


→ main에서 출발해 새 브랜치 생성
→ 이제 이 브랜치에서 마음껏 코드 수정

4. 작업 단위로 기록 (Commit)

VS Code에서 변경사항 확인

Staging:

git add .


Commit (메시지 포함):

git commit -m "feat: Double Bottom 매매전략 추가"

5. 원격 저장소에 반영 (Push)
git push origin feature/double-bottom-strategy


→ GitHub 웹사이트에서도 브랜치가 생긴 걸 확인할 수 있음

6. 병합 (Merge)

GitHub 웹에서 Pull Request(PR) 생성 → feature를 main에 병합

혼자 작업하셔도 PR → Merge 버튼 클릭 습관을 들이시면 좋습니다.
(이게 업계 표준 프로세스라, 나중에 협업할 때 바로 통합니다)

7. 정리

다음 작업 시작 시, 다시 main에서 출발 → 새로운 feature 브랜치 생성

즉,

main = 항상 안정된 버전

feature/… = 실험/개발 공간
